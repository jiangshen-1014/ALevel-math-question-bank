## Bug修复：\hfill 等文本模式 LaTeX 指令原样显示（22:45）
- **现象**：用户在题干写 `\hfill`，网页显示成字面文本 `\hfill`，未被渲染。
- **根因**：`\hfill` 是 LaTeX **文本模式**指令；MathJax 只处理 `$...$` 内的内容，题干里的裸 `\hfill` 不在数学区，MathJax 根本不碰；即便放在 `$...$` 内，默认 MathJax 也不定义 `\hfill`。
- **修复**（app.js `mdToHtml` 重写 + index.html MathJax 配置 + style.css）：
  1. `mdToHtml` 新增「数学区保护」：先把 `$...$`/`$$...$$` 抽成占位符，再对**正文文本**做 LaTeX 排版指令→HTML 转换，最后还原数学区。避免误伤数学内的反斜杠。
  2. 文本模式指令映射：`\hfill`→`<span class="hfill-line">` 内用 `flex; justify-content:space-between` 把两侧内容左右推开（一行内任意多个 `\hfill` 均分）；`\\`→`<br>`（排除 `\[ \] \( \)`）；`\newline`→`<br>`；`\noindent`→删；`\quad`→2空格、`\qquad`→4空格。
  3. index.html MathJax `macros` 新增 `hfill: ['\\hspace{\\fill}', 0]`，使 `$...$` 内的 `\hfill` 也不报错（给出弹性间隔）。
  4. style.css 新增 `.hfill-line { display:flex; justify-content:space-between; align-items:baseline; gap:1em; width:100% }` 及子 `span{min-width:0}`。
- **验证**：Node 单测 6 例全过（裸 \hfill→flex、数学区 `$x<3$` 不被转义、math 保留、`\\`→br、`\[ \]` 安全、quad 空格数、tikz 块仍正常）；app.js/data.js `node --check` 通过。
- 结论：现在 `\hfill` 在题干正文（脱离 `$...$`）也能正确渲染为左右对齐；TikZ/markdown/数学区均不受影响。
